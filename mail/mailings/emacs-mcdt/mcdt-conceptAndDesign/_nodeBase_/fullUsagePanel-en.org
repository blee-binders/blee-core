** Overview And Direction:
There are two layers 
1) mcdt-if.el  --- User Interface (currently mcdt.el)
2) mcdt-lib.el   --- Core -- base functions currently /bisos/blee/env//boot//bystar-mail-lib.el (revisit name)
User documentation is primarily limited to mcdt-if.el

The primary abstractions are:
- A Mailing -- Mailing-File
  A Mailing is a read-only static representation of parameters of an outgoing email. It is a single file.
- An Unsent-Mailing --
  An Unsent-Mailing is a buffer generated from a Mailing-File that can be sent.
- External-Source-Content -- LaTeX-Body

The primary argument to most functions are
1) <mailingFilePath
2) <mailingBuf
3) <mailingBasePath
4) <mailingName -- Based on X-Mailing-Name:
5) <mailingParams -- Based on X-Mailing-Params:
   org-dynamic-block-insert-dblock gets its parameters
5.1) :extSrc "." -- :extSrc nil
5.2) :type 'compose | 'redraft | 'static | 'dynamic

The following mailing commands are defmacro generated:

1) mcdt:compose/mailingName    -- Creates a compose buffer
2) mcdt:redraft/mailingName    -- subjetc to ctl u edits the current "unsent mail"
3) mcdt:batch/mailingName
4) x bbdb
5) web-url web-mailto
